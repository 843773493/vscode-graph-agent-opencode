# 目录用途

`scripts/` 存放仓库级开发、构建、生成、调试和轻量测试入口脚本。

# 可修改内容

- 可以新增或维护由 `package.json`、测试命令或开发流程调用的脚本。
- 可以放置 ESM `.mjs` 脚本，用于编排已有工具、生成类型或执行无 DOM 的状态测试。

# 不可修改内容

- 不在这里实现后端业务逻辑、前端运行时代码或持久化核心规则。
- 不放需要长期维护的大型测试数据、构建产物或运行时缓存。

# 规范

- JavaScript 脚本使用 ESM，通过 `import`/`export` 或顶层脚本逻辑组织，不使用 CommonJS。
- 脚本应从仓库根目录或显式传入路径解析文件，不通过脚本文件位置向上猜测项目根目录。
- 长命令优先收敛到脚本中，脚本失败时直接抛出明确错误。
- 保留“模板示例；在整理 `AGENTS.md` 时请保留此行。”这一行。
