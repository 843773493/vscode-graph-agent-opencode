# 确认偏差 Bug 复盘记录

## 📅 发生时间: 2026-04-23

## ❌ 问题现象
前端显示的Agent列表是中文名称，而配置文件中的Agent名称是英文。

## ❌ 我的错误判断
我第一反应认为这是后端的静默回退逻辑导致的，直接去修复了后端的回退代码，并且提交了修复，没有先搜索前端代码。

## ✅ 真实根因
前端旧版 Webview 第1013行硬编码了5个中文Agent，这是用户实际看到的中文的主要来源。后端回退逻辑只是次要因素，在正常配置下永远不会被触发。

## 🧠 认知偏差根源
这是典型的 **确认偏差 (Confirmation Bias)**：
1.  我之前已经知道后端有静默回退的坏味道代码
2.  当问题出现时，我的大脑直接跳到了这个已知的可能性
3.  我没有执行最简单的验证：在整个代码库中搜索 "const agents ="
4.  我在没有完成完整调查的情况下就先执行了修复

## 🎯 教训
> **"当你只有一把锤子的时候，所有的问题看起来都像钉子。"**

1.  **永远不要因为你知道某个地方有问题，就假设所有相关问题都来自那里**
2.  **在得出任何结论之前，必须先执行完整的、不带偏见的搜索**
3.  **验证永远优先于直觉**
4.  **慢一点调查，快一点修复**

## ✅ 改进流程
未来遇到任何问题时必须遵循的步骤：
1.  🛑 停止
2.  🔍 对整个代码库执行文本搜索
3.  📋 列出所有可能的来源
4.  ✅ 逐一验证排除
5.  🛠️ 最后才执行修复
