# Subagent 盲点 Bug 复盘记录

## 📅 发生时间: 2026-04-23

## ❌ 问题现象
中文Agent来源问题中，Explore subagent给出的答案指出问题在后端，而真实问题在前端。

## ❌ Subagent 回答错误的根本原因

我给Subagent的提问方式本身就带有强烈的引导偏差：
> ❌ 我提问的原文："查找为什么界面显示的agent名称都是中文，而配置文件里是英文。"

✅ Subagent正确的理解了这个问题，并且正确的发现了 **后端确实存在这样的回退逻辑**。Subagent没有撒谎，它给出的答案是完全正确的。

🔴 **我的错误**：我没有告诉Subagent **同时检查前端代码**。我默认的假设了问题在后端，并且在提问时隐含了这个假设。

---

## 🎯 这不是Subagent的错误，这是提问者的错误

| 问题所在 | 说明 |
|---|---|
| ❌ **不是Subagent错误** | Subagent正确的回答了它被问到的问题 |
| ❌ **是我提问方式的错误** | 我没有给出完整的搜索范围，我隐含的假设了问题在后端 |
| ❌ **Subagent没有自由** | Subagent会严格按照你给出的搜索范围工作，它不会主动超出范围搜索 |
| ❌ **这是最容易犯的错误** | 提问者总是假设自己知道问题大概在哪里 |

---

## ✅ Subagent 使用黄金法则

### 🚨 永远不要告诉Subagent去哪里找问题

| ❌ 错误的提问方式 | ✅ 正确的提问方式 |
|---|---|
| "在后端查找XX问题" | "在整个代码库中查找XX问题" |
| "检查API接口为什么XX" | "查找整个项目中所有和XX相关的代码" |
| "修复session服务中的XX" | "找到XX问题的根因，在哪里就修复哪里" |

---

## 🎯 根本教训

> Subagent是放大镜，不是侦探。
>
> 它会完美的在你指定的范围内搜索你让它找的东西。但是它永远不会超出你指定的范围。
>
> 如果你给它的范围是错的，它会非常完美的在错误的范围内给出正确的答案。

---

## ✅ 永远有效的提问模板

永远用这个格式提问：
```
在整个代码库中查找 [问题描述]。
搜索所有文件，不要做任何假设。
找到真实的根因，不管它在什么位置。
```

---

## 📌 永久记忆点

> 最完美的Subagent回答，也无法弥补提问者一开始就指错了方向。
>
> 这不是Subagent的盲点，这是所有人类在使用助手时的共同盲点。
