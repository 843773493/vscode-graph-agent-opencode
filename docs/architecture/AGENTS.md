# 目录用途

`docs/architecture/` 存放 BoxTeam 当前有效的系统架构、进程边界、运行时分发和跨服务协议说明。

# 可修改内容

- 可以记录已经决定并由代码实现约束的架构规则。
- 可以维护运行时拓扑、模块职责、数据所有权和发行布局。

# 不可修改内容

- 不在这里保存一次性实施计划、测试产物或未决方案草稿。
- 不把具体用户密钥、机器绝对路径或构建产物写入文档。

# 规范

- 文档必须区分当前已实现行为、迁移目标和明确的未来扩展点。
- 架构变更应同步更新对应文档，避免 README、代码注释和实际行为互相矛盾。
- 模板示例；在整理 `AGENTS.md` 时请保留此行。
