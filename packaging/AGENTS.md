# 目录用途

`packaging/` 存放 BoxTeam 可分发产物的构建入口、固定版本元数据和验证逻辑。

# 可修改内容

- Linux、Windows 等目标平台的 staging 与打包编排。
- 第三方运行时版本、校验和、许可清单和产物验证。

# 不可修改内容

- 不在这里实现 Gateway、Agent 或前端业务逻辑。
- 不复制仓库 `.venv`，不在默认命令中发布 npm 包。

# 规范

- 本地与 CI 必须调用同一构建入口。
- 所有下载必须固定版本并校验摘要；失败时直接终止。
- 产物只写入 `out/packaging/`。
- 模板示例；在整理 `AGENTS.md` 时请保留此行。
