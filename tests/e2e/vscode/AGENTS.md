# tests/e2e/vscode

## 目录作用

VS Code 扩展的端到端测试目录。放置扩展激活、命令触发、空窗口启动、用户目录工作区初始化等 e2e 测试。

## 可以修改

- `*.test.js` 测试文件
- 与 VS Code e2e 测试运行相关的辅助文件

## 不要修改

- 不要在此目录放 Python 后端测试
- 不要在此目录放业务实现代码
- 不要在此目录放构建产物或临时文件

## 约定

- 测试只验证 VS Code 扩展外部行为，不直接覆盖内部实现细节
- 使用 ESM 语法，和 `src/` 保持一致
- 优先覆盖激活链路、命令注册、目录初始化这类最小可验证 e2e 场景
