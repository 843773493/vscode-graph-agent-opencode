# tests/e2e

本目录包含端到端测试与真实运行验证，用于覆盖 API、任务调度、Agent 执行、工具调用与工作区写入等关键链路。

## May Modify

- 端到端测试用例
- e2e 公共测试辅助工具
- 与 e2e 测试直接相关的 fixture、标记与配置

## Do Not Modify

- 生产代码实现
- 其他测试目录的公共约定，除非明确需要同步调整
- 与 e2e 无关的脚本化调试代码

## Conventions

- 优先使用 pytest 风格组织测试
- 真实模型依赖测试必须显式检查环境变量并在缺失时跳过
- 公共等待与断言逻辑应抽到共享 helper，避免重复
- 测试命名应清晰表达场景与预期结果
