# src/test

## 目录作用

VS Code 扩展的自动化测试目录。放置扩展启动、命令注册、视图注册等烟雾测试。

## 可以修改

- `*.test.js` 测试文件
- 与测试运行相关的辅助文件

## 不要修改

- 不要在此目录放业务实现代码
- 不要在此目录放构建产物或临时文件

## 约定

- 测试只验证扩展行为，不直接覆盖前端业务实现细节
- 使用 ESM 语法，和 `src/` 保持一致
- 新增测试时优先覆盖启动、命令注册、关键视图注册等最小可验证链路# src/test
