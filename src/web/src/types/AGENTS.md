# 目录用途

存放浏览器前端消费的 TypeScript 类型定义，包括手写前端视图类型和从后端协议生成的类型入口。

# 可修改内容

- 调整前端内部视图类型、后端 DTO 映射类型和类型导出。
- 在后端协议更新后同步生成类型文件。

# 不可修改内容

- 不在本目录实现运行时逻辑、API 调用或 UI 渲染。
- 不在手写类型中伪造后端不存在的字段。
- 不直接修改生成目录中的文件，除非正在执行类型生成流程或修复生成脚本产物。

# 规范

- 手写类型应保持与后端 schema 语义一致。
- 生成类型的修改需要确认来源和生成命令。
- 修改本目录后需要运行前端静态检查和 `bun run build`。
