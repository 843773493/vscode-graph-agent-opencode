# 目录用途

存放浏览器前端可复用的 React hook，负责把组件或 Provider 中的副作用流程拆成可维护的小单元。

# 可修改内容

- 新增或调整与会话、事件流、Agent State、消息动作等相关的 hook。
- 封装 React 状态更新、请求竞态处理和浏览器事件监听。

# 不可修改内容

- 不在本目录实现 UI JSX 组件。
- 不在本目录定义后端协议类型的权威来源。
- 不在 hook 中吞掉后端错误；需要把错误写入状态或继续抛出。

# 规范

- hook 应围绕清晰的副作用边界命名。
- 复杂状态变换优先复用 `state/` 下的纯函数。
- 修改本目录后需要运行前端静态检查和 `bun run build`。
