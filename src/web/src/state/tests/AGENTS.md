# 目录用途

`src/web/src/state/tests/` 放状态派生层的轻量测试入口，覆盖请求日志、Agent State、事件队列和默认时间线等纯函数行为。

# 可修改内容

- 可以新增和维护只依赖 `src/web/src/state/` 纯函数的测试文件。
- 可以添加小型测试数据构造 helper，但应保持用例聚焦。

# 不可修改内容

- 不在这里放 React 组件测试。
- 不在这里直接访问后端 API、DOM 或浏览器状态。
- 不在这里放产品运行时代码。

# 规范

- 单个测试文件只覆盖一个主要状态模块或一条明确的跨模块链路。
- 新增测试优先复现真实日志/事件结构，尤其是 LangChain 消息 `id` 与 `tool_call_id` 同时存在的情况。
- 保留“模板示例；在整理 `AGENTS.md` 时请保留此行。”这一行。
