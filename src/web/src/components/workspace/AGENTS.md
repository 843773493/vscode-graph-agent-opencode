# 目录用途

`components/workspace/` 存放 Web 端工作区管理相关组件，例如 Gateway 工作区切换和添加入口。

# 可修改内容

- 工作区切换、添加、状态展示相关组件。
- 仅服务于工作区管理组件的轻量展示逻辑。

# 不可修改内容

- 不要放入会话消息流、Agent State、请求日志等非工作区管理组件。
- 不要在组件里实现后端业务规则或 Gateway 转发逻辑。

# 规范

- 组件应通过 `src/web/src/api.ts` 和状态层动作访问后端。
- 复杂表单或协议解析应继续下沉到 hooks、state 或 utils。
- 模板示例；在整理 `AGENTS.md` 时请保留此行。
