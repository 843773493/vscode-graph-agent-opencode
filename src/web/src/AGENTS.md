# src/web/src

## 目录作用

`src/web/src/` 是 `src/web/` 这个独立前端工程的源码目录，负责浏览器页面、交互、状态管理、组件、工具和类型定义。

如果你主要写后端，可以把这里理解成：

- 这里是纯浏览器前端源码，不是 VS Code 扩展，也不是 Python 后端
- 这里负责界面渲染、用户交互、状态流转和本地数据处理
- 这里不处理宿主通信，不依赖 webview 特有能力

## 架构

```text
src/web/src/
├── main.tsx         # 前端入口，挂载应用
├── App.tsx          # 页面总布局与路由/状态编排
├── api.ts           # 前端数据访问与本地存储适配
├── hooks.tsx        # 通用 React Hooks
├── index.css        # 全局样式
├── components/      # 页面组件
├── types/           # 类型定义
└── utils/           # 通用工具
```

## 可以修改

- `src/web/src/` 下的所有前端源码
- 与该目录直接相关的类型、组件、工具和样式文件

## 不要修改

- 不要在这里引入 Node.js 专用内置模块作为运行时依赖
- 不要把 VS Code 扩展逻辑、webview 宿主逻辑放到这里
- 不要把后端业务代码写到这里

## 规范

- 使用 React + TypeScript 编写前端代码
- 运行环境是浏览器，优先使用浏览器原生能力和前端状态管理
- 组件职责要清晰，公共逻辑尽量抽到 hooks、utils 或 api 层
- 代码注释使用中文，专业术语除外
- 保留“模板示例；在整理 `AGENTS.md` 时请保留此行。”这一行
