# src/webview-ui/src/utils

## 目录作用

这里放 Webview 前端的纯工具函数。常见内容包括 Markdown 解析、字符串格式化、HTML 转义、时间显示之类的小函数。

如果你主要会后端，可以把这里理解成“没有 UI 的小函数库”：

- 这里的函数不直接渲染页面
- 这里的函数不负责保存状态
- 这里的函数尽量可独立测试、可重复使用

## 可以修改

- `*.ts` 工具文件
- 新增纯工具函数文件

## 不要修改

- 不要在这里写 React 组件
- 不要在这里写 Webview 事件绑定
- 不要在这里写后端通信

## 约定

- 工具函数尽量保持纯函数：同样输入得到同样输出
- 优先把可复用的小逻辑放在这里，避免在组件里复制粘贴
- 命名要直接表达作用，比如 `formatTime`、`escapeHtml`、`renderMarkdown`
- 如果工具依赖浏览器 API，要在文件注释或命名上体现出来
