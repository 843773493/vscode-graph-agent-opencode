# src/webview-ui/src/components

## 目录作用

这里放的是 Webview 前端页面上的具体 UI 组件。每个文件通常负责一个页面区块，比如聊天列表、输入框、工具栏、历史面板。

如果你主要会后端，可以把这里理解成“页面上的零件”：

- 组件负责展示和局部交互
- 组件之间尽量少互相直接调用
- 状态和消息流优先放到 `src/hooks.tsx` 管

## 可以修改

- `*.tsx` 组件文件
- 组件内部样式相关的小逻辑

## 不要修改

- 不要在组件里放共享协议定义
- 不要在组件里写 Node.js / 后端调用逻辑
- 不要把全局状态散落到每个组件里

## 约定

- 一个组件尽量只做一件事
- 可以复用的展示逻辑优先抽成小组件
- 可以复用的交互逻辑优先抽成 hook 或 helper
- 组件间的数据流优先从上层传 props，不要绕过状态中心直接改全局数据
