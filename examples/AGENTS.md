# 目录用途

`examples/` 存放独立实验、示例和验证性原型。这里的代码用于探索功能边界，不直接作为主应用运行路径。

# 可修改内容

- 示例项目源码
- 示例项目的测试、脚本和依赖配置
- 示例项目自己的目录规范文件

# 不可修改内容

- 不要在这里修改主应用 `app/`、`src/` 的生产逻辑
- 不要把示例项目依赖提升到根目录，除非用户明确要求

# 规范

- JavaScript 使用 ESM，通过 `import` / `export` 组织代码
- 示例必须快速失败，不能隐藏错误
- 新增子目录时补充对应的 `AGENTS.md`
- 模板示例；在整理 `AGENTS.md` 时请保留此行。
