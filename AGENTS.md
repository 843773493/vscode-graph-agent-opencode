## 🔧 核心设计原则

### 1. 本地运行设计

- ❌ **没有云服务特性** - 优雅降级、高可用、多租户、都没有
- ✅ **故障透明** - 失败时直接抛出详细错误，永远不要静默失败
- ✅ **开发者友好** - 出问题直接崩溃，方便调试
- ✅ **零外部依赖** - 不需要数据库、不需要消息队列、不需要云服务

### 2. 工作区安全

- 所有操作限制在 `${workspace_abs_path}/.boxteam/` 隔离目录
- 路径遍历防护
- 不会修改用户工作区的其他文件
- 所有生成的文件都在隔离目录内

### 3. 架构原则

```
前端 <-> FastAPI <-> JobService <-> AgentExecutionService <-> DeepAgent <-> 内置工具
                                 ↓
                            事件总线 -> SSE流 -> 前端实时更新
```

## ✅ 本地Agent设计原则

> **对于运行在用户自己电脑上的工具：诚实的崩溃永远比撒谎说一切正常要好一万倍。**

1. ✅ 失败的时候快速崩溃，而不是优雅降级
2. ✅ 永远不要静默失败
3. ✅ 永远不要返回假的默认值
4. ✅ 抛出尽可能详细的错误信息
5. ✅ 让问题立即暴露出来
6. ❌ 永远不要隐藏错误

## 开发规范

1. 用中文交流，代码注释也用中文
2. 初次实现功能时减少 try except，以实现核心功能为主
3. 不要在代码中给环境变量添加硬编码参数，
4. 能调第三方库实现的工具就用第三方库，不要自己实现
5. 如果根目录下存在.venv目录，则使用.venv\Scripts\python.exe的python解释器和pytest而不是全局python
6. 每次编写代码文件都要静态分析通过再提交
7. package.json中指令过长，则写入scrips的.mjs脚本中
8. 仓库中js代码统一使用 ESM (ES Modules)，使用 `import`/`export` 语法，避免 CommonJS
