# BoxTeam - VS Code AI 编程助手

## 🎯 项目目标

这是一个运行在用户本地工作区的AI编程助手后端，配合前端VS Code扩展实现IDE级别的自动编程体验。

---

## 🔧 核心设计原则

### 1. 本地运行设计

- ❌ **没有云服务特性** - 优雅降级、高可用、多租户、都没有
- ✅ **故障透明** - 失败时直接抛出详细错误，永远不要静默失败
- ✅ **开发者友好** - 出问题直接崩溃，方便调试
- ✅ **零外部依赖** - 不需要数据库、不需要消息队列、不需要云服务

### 2. 工作区安全

- 所有软件数据储存在 `${workspace_abs_path}/.boxteam/` 独立目录

### 3. 架构原则

```
前端 <-> FastAPI <-> JobService <-> AgentExecutionService -> DeepAgent -> 内置工具
                                  ↓
                             事件总线 -> SSE流 -> 前端实时更新
```

### 4. 前端状态管理原则

> **后端为单一数据源（Backend-First State Management）**

前端不维护业务状态的"真实来源"，所有状态变更必须通过后端API，成功后**以后端返回的完整对象完全替换前端状态**（而非部分修改字段），失败时主动从后端重拉数据以保证一致性。适用于Agent切换、Session管理、消息发送等核心业务状态。

## 🚀 运行方式

```bash
# 安装依赖
.\.venv\Scripts\pip.exe install -r requirements.txt

# 启动后端服务器
.\.venv\Scripts\uvicorn.exe app.main:app --host 127.0.0.1 --port 8000
```

API文档: 👉 http://127.0.0.1:8000/api/v1/docs

---

## ✅ 本地Agent设计原则

> **对于运行在用户自己电脑上的工具：诚实的崩溃永远比撒谎说一切正常要好一万倍。**

1. ✅ 失败的时候快速崩溃，而不是优雅降级
2. ✅ 永远不要静默失败
3. ✅ 永远不要返回假的默认值
4. ✅ 抛出尽可能详细的错误信息
5. ✅ 让问题立即暴露出来
6. ❌ 永远不要隐藏错误

---

## 开发规范

1. 用中文交流，代码注释也用中文
2. 初次实现功能时减少 try except，以实现核心功能为主
3. 尽量避免使用any类型，除非遇到泛型等复杂情况，可以保留any
4. 不要在代码中给环境变量添加硬编码参数，
5. 能用第三方库就添加并使用第三方库，不要重复造轮子
6. 非用户主动要求，不要写文档
7. 如果根目录下存在.venv目录，则使用.venv\Scripts\python.exe的python解释器和pytest而不是全局python
8. **前端目录**负责页面、交互、状态、接口调用、少量展示层逻辑。
9. **后端目录** 负责业务规则、权限、数据库、订单流转、风控、核心计算。
10. 懒加载的包单独放runtime模块里
11. 类：禁止原型混入/变异。优先继承/组合
12. 每次编写代码文件都要静态分析通过再提交
13. package.json中指令过长，则写入scrips的.mjs脚本中
14. 仓库中js代码统一使用 ESM (ES Modules)，使用 `import`/`export` 语法，避免 CommonJS
15. git提交：常规风格、简洁、分组。
